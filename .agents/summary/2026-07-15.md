# amatiasq.com — Resumen de exploración (2026-07-15)

## Qué es

El portfolio profesional de A. Matías Quezada: blog, experiencia laboral
(timeline), proyectos, charlas y experimentos. Sitio **estático** generado con
**Astro 5**, servido en `https://amatiasq.com`. Bilingüe (en / es), pensado para
sumar catalán. ~2.900 líneas.

- **Runtime**: Bun (hay `bun.lock`, la CI usa `oven-sh/setup-bun`).
- **Contenido**: markdown en `content/` (blog, career, projects, talks,
  experiments), cargado por colecciones de Astro con schemas Zod.
- **Sin JS de cliente** salvo dos scripts mínimos (pausar un iframe, copiar
  bloques de código). Fiel a su propio AGENTS.md: "JavaScript mínimo".

## Cómo está organizado

Atomic design estricto: `atoms/` (Image, Link, Time, Tags, Translation…),
`molecules/` (Markdown), `organisms/` (CareerTimeline, ContentCards,
NavHeader…). Las páginas viven en `src/pages/[lang]/<colección>/`, con un
`index.astro` (lista) y un `[slug].astro` (detalle) por colección.

## Flujo de datos

```
content/<colección>/*.md
   │  (frontmatter Zod + body partido por `---` = un bloque por idioma)
   ▼
content.config.ts  ── loader markdown propio ──►  colecciones de Astro
   │
   ▼
pages/[lang]/<colección>/[slug].astro
   │  useTranslations(url) → tr()      (elige idioma por la URL)
   │  body[languages.indexOf(lang)]    (elige el bloque de idioma)
   ▼
Layout.astro  →  componentes (Markdown, Image, Tr…)  →  HTML estático
```

La internacionalización es el eje: cada texto traducible es
`{ en, es }` o `[en, es]`, y `tr()` lo resuelve según el idioma de la URL. El
`body` del markdown se parte por `---` en un bloque por idioma.

## Estado de cada zona

| zona | qué hace | estado |
| --- | --- | --- |
| `i18n.ts` | i18n funcional basado en URL (`useTranslations`) | sólido, es el sistema real |
| `internals/i18n.ts` | **segundo** i18n, con estado global mutable | usado en un solo sitio, y roto (ver abajo) |
| `content.config.ts` | schemas Zod + loader markdown propio (180 líneas) | los schemas, bien; el loader es **necesario** (parte el body por idioma, algo que el `glob()` oficial no deja hacer) pero su comentario esconde el porqué |
| `Layout.astro` | shell HTML + todo el CSS global (los estilos base viven aquí) | correcto; el CSS es grande pero es la hoja del sitio |
| `Markdown.astro` | markdown → HTML con resaltado Shiki | funciona, pero recrea el highlighter en cada render |
| `CareerTimeline.astro` | timeline SVG-like de experiencia | el más grande (422 líneas), `any` por todas partes, colores de marca incrustados |
| páginas `[slug]` | detalle de cada colección | correctas, pero repiten el mismo bloque de localización 6 veces |
| `util/date.ts` | parseo y formato de fechas parciales (`2021`, `2021-05`) | limpio y bien tipado |

## Deuda técnica y hallazgos

1. **Dos sistemas de i18n, y el segundo está roto.** `internals/i18n.ts`
   guarda el idioma en un `let currentLang` de módulo que **nadie inicializa**
   (`setLanguage` no se llama en ningún sitio), así que siempre vale `'en'`. Lo
   usa sólo `Image.astro`, de modo que una imagen con `src`/`alt` traducible
   siempre sale en inglés. Es a la vez un bug latente y justo lo que el AGENTS.md
   del mono prohíbe ("no loose module-level variables").

2. **Shiki se recrea en cada `<Markdown>`.** `createHighlighter` con 15
   lenguajes corre una vez por cada render del componente; con ~24 posts × 2
   idiomas + proyectos, son decenas de inicializaciones en cada build. Se crea
   una vez y se reutiliza.

3. **Duplicación de la localización del body.** Las mismas dos líneas
   (`Array.isArray(body) ? … : [body]` + `content[languages.indexOf(lang)]`)
   aparecen idénticas en las 6 páginas `[slug]`.

4. **`any` a discreción** (45 apariciones). `CareerTimeline` y `ContentCards`
   reciben `any[]` y mapean con `(item: any)`. El AGENTS.md del mono pide
   preguntar antes de usar `any`.

5. **El loader markdown propio** (180 líneas en `content.config.ts`, rotulado
   "pay no attention to it") existe por una razón legítima y no reemplazable:
   parte el body por `---` en un bloque por idioma, y el `glob()` oficial de
   Astro 5 no da ningún hook para tocar el body. El problema no es el loader, es
   que el comentario esconde el motivo — invita a borrarlo por parecer copia
   gratuita cuando es justo lo que hace posible el markdown multilingüe.

## Inconsistencias

- **README desactualizado**: dice `npm install` / `npm run dev`, pero el
  proyecto va con Bun (lockfile y CI).
- **Colores de marca** de empresas incrustados en `CareerTimeline` — datos de
  contenido dentro de un componente de presentación.
- **Warnings de `astro check`** ya presentes: `frameborder` deprecado (×2), un
  `tr` declarado sin usar en la página de tags.
